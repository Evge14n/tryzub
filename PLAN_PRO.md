## БЛОК 1: Stack traces та професійні помилки

### 1.1 Call stack tracking
- В VM додай `call_stack: Vec<CallFrame>` де `CallFrame { function_name: String, file: String, line: usize }`
- При кожному виклику функції — push, при return — pop
- При помилці — друкуй весь стек:
```
паніка: ділення на нуль
  → обчислити_податок() в податки.тризуб:42
  → обробити_замовлення() в замовлення.тризуб:15
  → головна() в програма.тризуб:3
```

### 1.2 Розумні повідомлення про помилки
- Додай "Did you mean?" для невідомих змінних/функцій (Levenshtein distance ≤ 2)
- Додай error codes: Т001 (невідома змінна), Т002 (типова невідповідність), тощо
- При type mismatch: `очікувався тип 'ціле', отримано 'рядок' в рядку 15`
- Підказки: `змінна х: ціле = "hello"  // підказка: можливо ви мали на увазі ціле_з_рядка("hello")?`

### 1.3 Integration test
```rust
#[test]
fn test_stack_trace() {
    // Виклик функції що панікує → перевірити що stack trace містить всі фрейми
}

#[test]
fn test_did_you_mean() {
    // Помилка в імені змінної → перевірити що підказка з'являється
}
```

## БЛОК 3: Генерики

### 3.1 Мономорфізація
```
функція макс<Т>(а: Т, б: Т) -> Т {
    якщо а > б { а } інакше { б }
}

друк(макс(5, 3))        // макс_ціле
друк(макс(1.5, 2.7))    // макс_дробове
друк(макс("а", "б"))    // макс_рядок
```
- При виклику — визначити конкретний тип Т з аргументів
- Створити спеціалізовану версію функції
- Кешувати спеціалізації

### 3.2 Обмеження на генерики (trait bounds)
```
функція сортувати<Т: Порівнянне>(масив: [Т]) -> [Т] { ... }
```

### 3.3 Генеричні структури
```
структура Пара<А, Б> {
    перше: А,
    друге: Б
}
```

## БЛОК 4: Трейти з динамічним dispatch

### 4.1 Визначення та реалізація
```
трейт Друкований {
    функція в_рядок(себе) -> рядок
}

реалізація Друкований для Точка {
    функція в_рядок(себе) -> рядок {
        повернути ф"{себе.х}, {себе.у}"
    }
}
```

### 4.2 Dynamic dispatch
```
функція друкувати_все(елементи: [Друкований]) {
    для е в елементи {
        друк(е.в_рядок())
    }
}
```
- Реалізувати vtable для кожного типу що реалізує трейт
- Зберігати вказівник на vtable разом з об'єктом

### 4.3 Default methods
```
трейт Порівнянне {
    функція менше(себе, інше) -> логічне

    функція більше(себе, інше) -> логічне {
        повернути інше.менше(себе)
    }
}
```

### 4.4 Operator overloading через трейти
```
трейт Додавання<Інше = Себе> {
    функція додати(себе, інше: Інше) -> Себе
}

реалізація Додавання для Вектор {
    функція додати(себе, інше: Вектор) -> Вектор {
        повернути Вектор { х: себе.х + інше.х, у: себе.у + інше.у }
    }
}

змінна а = Вектор { х: 1, у: 2 }
змінна б = Вектор { х: 3, у: 4 }
змінна с = а + б  // викликає а.додати(б)
```

## БЛОК 5: Ліниві ітератори

### 5.1 Протокол ітератора
```
трейт Ітератор<Т> {
    функція наступний(себе) -> Т?  // повертає Деякий(значення) або Нічого
}
```

### 5.2 Ланцюжки
```
змінна результат = діапазон(1, 1000000)
    |> фільтрувати(|х| х % 2 == 0)
    |> перетворити(|х| х * х)
    |> взяти(10)
    |> в_масив()
```
- НЕ створювати проміжні масиви
- Кожен елемент проходить весь ланцюг по одному
- `взяти(10)` зупиняє ітерацію після 10 елементів

### 5.3 Генератори як ітератори
```
функція фібоначчі() -> Ітератор<ціле> {
    змінна а = 0
    змінна б = 1
    поки істина {
        віддати а
        змінна т = а + б
        а = б
        б = т
    }
}

для ч в фібоначчі().взяти(20) {
    друк(ч)
}
```

## БЛОК 6: Async/Await

### 6.1 Event loop
```
асинхронна функція завантажити(url: рядок) -> рядок {
    змінна відповідь = чекати http_отримати(url)
    повернути відповідь
}

асинхронна функція головна() {
    змінна сторінки = чекати все([
        завантажити("https://example.com/1"),
        завантажити("https://example.com/2"),
        завантажити("https://example.com/3")
    ])
    друк(сторінки)
}
```
- Інтегрувати tokio або написати простий event loop
- `чекати` — suspend і resume
- `все([...])` — паралельне виконання
- `перегони([...])` — перше що завершилось

### 6.2 Канали
```
змінна (відправник, отримувач) = канал()

потік(|| {
    відправник.надіслати("привіт")
})

змінна повідомлення = отримувач.отримати()
```

## БЛОК 7: Базова тип-система

### 7.1 Type checking на етапі компіляції
```
змінна х: ціле = "hello"  // ПОМИЛКА: очікувався тип 'ціле', отримано 'рядок'

функція додати(а: ціле, б: ціле) -> ціле {
    повернути а + б
}
додати("не", "число")  // ПОМИЛКА: аргумент 1: очікувався 'ціле', отримано 'рядок'
```

### 7.2 Type inference
```
змінна х = 42       // автоматично: ціле
змінна у = "текст"  // автоматично: рядок
змінна з = х + 1    // автоматично: ціле
```

### 7.3 Union types
```
змінна значення: ціле | рядок = 42
значення = "текст"  // ОК
значення = істина   // ПОМИЛКА
```
